# 太阳能草坪灯：接口与逻辑

> 目标：把"光控（4）+ICM7555时控（5）+逻辑控制（6）"三块串成一个稳定的自动循环：
> **天黑自启 → 计时照明 → 到时关断并夜间锁死 → 天亮复位**。

## 1. 信号与接口定义（建议统一命名）

### 1.1 电源与模拟量
- **V_BAT**：电池端电压（2S锂电 7.4–8.4V）。
- **GND**：系统地。
- **V_LDR**：光敏分压节点电压（随照度变化）。
- **V_REF_H / V_REF_L**：比较器参考阈值（可调）；配合迟滞形成开/关两个阈值区间。

### 1.2 数字/控制信号
- **DARK**：光控判据输出（定义：天黑=1，天亮=0）。
- **DAY**：天亮信号
- **TRIG_555**：触发ICM7555单稳态的短脉冲（由"天黑沿"产生）。
- **T_ON**：ICM7555输出（计时有效期为1，到时为0）。
- **LOCK**：夜间锁死标志（1=禁止再次触发；0=允许触发）。
- **LED_EN**：LED驱动使能（最终去控制MOS栅极/驱动IC）。

## 2. 光控检测（LDR + 比较器 + 迟滞）

### 2.1 输出目标（任务书指标映射）
- **开启阈值**：2–10 lux 可调（对应 DARK 从0翻到1）。
- **关断阈值**：20–50 lux 可调（对应 DARK 从1翻到0）。
- **响应时间**：<2 s。
- **稳定性**：昼夜交替不抖动误触发（靠迟滞实现）。

### 2.2 关键电路思想
- LDR 与电阻分压得到 **V_LDR**。
- 低功耗比较器将 V_LDR 与参考比较，输出 **DARK**。
- 在比较器输出到输入侧引入正反馈形成**迟滞**：
  - 当环境从亮→暗时，阈值为 V_TH_ON（对应2–10 lux）；
  - 当环境从暗→亮时，阈值为 V_TH_OFF（对应20–50 lux）。

### 2.4 阈值与迟滞怎么“算出来”

#### 2.4.1 把照度映射到电压：V_LDR 的表达式
“LDR在上、定值电阻R_B在下”的分压：

$$V_{LDR} = V_{CC}\cdot \frac{R_B}{R_B + R_{LDR}(lux)}$$

- 天越黑，$R_{LDR}$ 越大，则 $V_{LDR}$ 越小。


#### 2.4.2 两个阈值如何实现：用施密特迟滞得到 V_TH_ON / V_TH_OFF
迟滞的本质是：输出高/低时，比较器的等效参考电压不同。

以“参考端由分压 R1/R2 产生，并通过 R_FB 从输出回灌”的典型结构为例，可得到：
- 输出为低时的参考：$V_{REF\_L}$
- 输出为高时的参考：$V_{REF\_H}$

然后定义：
- 若用 $V_{LDR}$ 与 $V_{REF}$ 比较，则
  - **开启点（亮→暗）**满足 $V_{LDR} = V_{REF\_H}$（或 $V_{REF\_L}$，取决于输入极性）
  - **关断点（暗→亮）**满足 $V_{LDR} = V_{REF\_L}$（或相反）


### 2.5 为什么要考虑温度补偿
- LDR、分压电阻、比较器输入失调都会随温度漂移，-20℃~+60℃下会导致阈值偏移。
- 若不加处理，可能出现“冬天提前亮/夏天延后亮”等现象，甚至在临界照度附近误动作。

**简化可行的补偿思路（按复杂度从低到高）：**

- **先用迟滞兜底**：迟滞本身能覆盖一部分温漂造成的抖动，这是最低成本最有效的一步。
- **参考端引入NTC微调温漂**：在 $V_{REF}$ 生成网络里串/并一个NTC，让参考电压随温度变化方向与LDR漂移方向部分抵消。
- **器件选型降低温漂源头**：关键电阻选低温漂、比较器选低失调/低漂移型号。

### 2.6 为什么分压要用“大电阻”
- 控制电路静态电流预算极小，光控分压若用 10k/10k 这类常规值，单分压就会消耗 $I=V/R$ 级别的电流。
- 用百k到MΩ量级电阻，能把分压电流压到微安级甚至更低。
- 电阻越大，节点阻抗越高，越容易受噪声/漏电影响，所以要配合：输入端小电容滤波、比较器输入保护、以及迟滞量设计。

### 2.7 实施要点（落地检查项）
- 分压网络优先使用较大阻值（降低静态电流），并在仿真/实测验证噪声抗扰。
- 阈值可调实现优先级：
  1) 原型阶段用电位器快速定标；
  2) 定型后改为“固定电阻+可选档位/替换1–2个元件”满足任务书要求。
- 输出端建议增加适度的去抖/滤波（例如在比较器输入端加小电容形成 <2s 的响应约束内的滤波），避免LED开关瞬态耦合导致的误触发。

### 2.8 “开启/关断阈值独立可调”：双比较器 + RS锁存

> “天黑触发”是 **V_LDR 下降穿越开启阈值**，“天亮关断/复位”是 **V_LDR 上升穿越关断阈值**。

工程上最清晰、可解释性最强、调试最方便的方式是：

**（1）用两个比较器分别产生 SET（变黑）与 RESET（变亮）事件**
- 比较器 A（Dark-Set 比较器）：
  - 输入：V_LDR
  - 参考：V_ON（开启阈值对应电压，可调）
  - 输出逻辑：当 **V_LDR < V_ON** 时输出有效（表示“足够暗，可以置位DARK”）
- 比较器 B（Day-Reset 比较器）：
  - 输入：V_LDR
  - 参考：V_OFF（关断阈值对应电压，可调）
  - 输出逻辑：当 **V_LDR > V_OFF** 时输出有效（表示“足够亮，可以复位DARK”）

这样就拥有两只旋钮/两组电阻：
- **V_ON** 调“变暗到何时开灯”
- **V_OFF** 调“变亮到何时关灯/复位”

**（2）用RS锁存把两个事件变成稳定的状态信号 DARK**
- 令：
  - S = (V_LDR < V_ON)
  - R = (V_LDR > V_OFF)
- RS锁存输出 Q 作为 DARK（天黑=1）

这一步的意义（也是写论文要强调的原因）：
- 比较器本质输出的是“是否越过阈值”，而RS锁存提供了“记忆”，让系统状态在阈值区间内不会抖动。
- 开启与关断阈值完全独立，满足任务书“独立可调”的表述，比单比较器施密特更容易写清楚与调准。

**（3）器件实现（数字/模拟混合）**
- 两比较器：优先选双比较器（如LM393一类）或超低功耗比较器。
  - LM393类常见为开集电极输出，需要上拉电阻。
- RS锁存：
  - 用两只施密特门交叉耦合构成SR锁存，抗干扰强；

**（4）阈值“lux→电压”落地到 V_ON/V_OFF**
- 用 LDR 曲线把目标照度对应到 $R_{LDR}$，再用分压式算出 V_LDR，即：

$$V_{LDRnode}=V_{CC}\cdot\frac{R}{R+R_{LDR}}$$

- 用 2–10 lux 得到一组 V_ON 范围；用 20–50 lux 得到一组 V_OFF 范围。
- 然后把 V_ON、V_OFF 分别用“电位器+限位电阻”或“可替换1–2个电阻”的方式产生。

### 2.9 用D触发器替代RS锁存

相比“裸RS锁存”，D触发器方案更容易做到：
- 上电状态确定（配合POR）
- 复位优先规则明确
- 论文中状态方程表达更规范

#### 2.9.1 逻辑定义
- 仍使用双比较器产生：
  - **SET_EVT**：当 $V_{LDR}<V_{ON}$（足够暗）时有效
  - **RST_EVT**：当 $V_{LDR}>V_{OFF}$（足够亮）时有效
- 状态位：$Q=DARK$（天黑=1，天亮=0）

状态方程：

$$Q^+ = SET\_EVT \lor \big(Q \land \lnot RST\_EVT\big)$$

该方程含义：
- 暗事件来时置1；
- 亮事件来时清0；
- 两事件都无效时保持原状态。

#### 2.9.2 D触发器实现思路

思路：把“状态方程”做成D端组合逻辑，再用事件脉冲更新Q。

1) **D端逻辑网络**
- 用门电路实现上式，得到 D_IN。

2) **CLK事件脉冲**
- 将 SET_EVT 与 RST_EVT 通过“或”合成为 EVENT；
- 对 EVENT 做整形（短脉冲）后送到 CD4013 的 CLK；
- 每次发生阈值事件时，Q按D_IN更新一次。

3) **异步复位（POR）**
- 上电时用RC在 CD4013 的 CLR 端施加有效复位，强制 $Q=0$（白天态）。
- POR释放后系统进入正常事件驱动。

4) **异步端优先级建议**
- 若同时担心竞争，建议把“亮复位”并入异步CLR路径（通过门控），形成**复位优先**。

#### 2.9.3 与ICM7555触发的接口

无论采用上述哪种CD4013接法，输出Q都可作为 DARK 状态源。

- 先由 Q 的上升沿（DARK上升沿）经 RC 微分 + 钳位得到 TRIG_555；
- 再与 DUSK（天黑过程）和非锁死条件门控：

$$TRIG\_{555}=pulse(Q\uparrow)\land DUSK\land(\lnot LOCK)$$

这样既满足“只在进入天黑时触发一次”，又避免夜间重复触发。

#### 2.10.1 信号标准化（先统一为高有效逻辑）
- COMP_DARK：暗阈值比较器输出，高有效，表示 $V_{LDR}<V_{ON}$。
- COMP_BRIGHT：亮阈值比较器输出，高有效，表示 $V_{LDR}>V_{OFF}$。
- DUSK：天黑过程检测，高有效（由快慢比较器得到“变暗”）。
- POR：上电复位信号，高有效。

比较器是开集电极输出，先做上拉，再按需要反相得到上述“高有效”定义。

#### 2.10.2 复位优先仲裁逻辑（核心）
- 原始请求：

$$S_{raw}=COMP\_DARK\land DUSK$$
$$R_{raw}=COMP\_BRIGHT\lor POR$$

- 仲裁后送锁存器：

$$R=R_{raw}$$
$$S=S_{raw}\land \lnot R_{raw}$$

这样可保证：当置位和复位请求同时出现时，复位优先，S被屏蔽，不会进入未定义输入组合。

#### 2.10.3 锁存实现（异步，无时钟）
- 方案A：两级NAND构成SR锁存（推荐配施密特输入门）。
- 方案B：D触发器的异步端构造锁存效果（PRE/CLR，按器件有效电平接入）。

无论A/B，都按 2.12.2 的 S、R 连接，确保“结构性无非法输入”。

#### 2.10.4 上电确定态（POR）
- POR直接进入复位链路（并入 R_raw），使系统上电必定进入 DARK=0。
- POR时间覆盖“电源爬升 + 比较器稳定”窗口；优先推荐电源复位芯片，RC次之。

#### 2.10.5 与ICM7555接口
- 从 DARK 上升沿取脉冲：RC微分 + 二极管钳位整形。
- 最终触发：

$$TRIG_{555}=pulse(DARK\uparrow)\land (\lnot LOCK)$$

若要更保守，可额外门控 DUSK：

$$TRIG_{555}=pulse(DARK\uparrow)\land DUSK\land (\lnot LOCK)$$

#### 2.10.6 最小器件
- 双比较器：1片（暗阈值、亮阈值）
- 趋势比较器：1片（或复用双比较器剩余通道）
- 逻辑门：与门、或门、反相门各1路（可用一片多门CMOS）
- 锁存单元：1路（NAND锁存或D触发器异步端）
- POR：RC或复位芯片1路
- ICM7555触发整形：R_TRIG、C_TRIG、钳位二极管

用这套结构，可以在无时钟条件下得到可证明的确定初态和确定优先级，同时保持任务书要求的独立阈值调节能力。

## 3. 模块5：时控（ICM7555 单稳态 + 多档时间）

### 3.1 输出目标
- 多档定时。

### 3.2 关键电路思想
- 选用 **ICM7555**（CMOS 555，静态电流更低）。
- 单稳态：被 **TRIG_555** 触发后，输出 **T_ON=1**，持续时间约：

$$T \approx 1.1RC$$

- 多档时间：固定 C，切换 R（拨码/旋转开关）。

### 3.3 为什么必须是 ICM7555（而不是传统NE555）
- 传统双极型NE555静态电流通常远高于该量级（具体以数据手册为准），会让系统在白天/关灯状态仍持续耗电。
- ICM7555（CMOS 555）的优势是静态电流更低、输入阻抗高，更适合"长时间待机+偶尔工作"的太阳能灯场景。

### 3.4 多档定时怎么设计成“可计算+可校准”
把定时设计写成三步：
1) 选定电容 C（优先低漏电、温漂小的类型；容量越大，对漏电/容差越敏感）。
2) 用 $T\approx 1.1RC$ 计算各档目标电阻：

$$R = \frac{T}{1.1C}$$

3) 选取标准阻值并建立“档位—目标时间—仿真时间—实测时间”的对照表。

### 3.5 为什么长延时容易不准

- **电容容差**：电解/钽等大容量器件容差往往较大。
- **电容漏电流**：长时间充电过程中，漏电会等效改变RC充电曲线，导致实际T偏差。
- **温度系数**：-20℃~+60℃下C值和漏电都会变化，定时会漂。
- **ICM7555内部阈值偏差**：不同批次/型号的阈值电平（1/3、2/3 VCC附近）存在容差。


### 3.6 RESET 管脚（与模块4/6耦合点）
- ICM7555 的 **RESET 为低有效**。
- 推荐连接：**RESET = DARK**（或等效门控）
  - 白天 DARK=0 → ICM7555强制复位，T_ON为0，避免白天误计时；
  - 夜间 DARK=1 → 允许触发与计时。

这么连的原因：
- 确保"天亮关断"：即便ICM7555处在计时中，只要天亮（DARK=0），立即复位输出为0，满足"天亮关断"的硬约束。
- 避免白天误触发：白天LDR边缘抖动也不会启动计时，减少无意义能耗。

### 3.7 TRIG_555 脉冲宽度与幅度为什么要整形
- ICM7555触发通常对脉冲的幅度/持续时间有最低要求（以所选ICM7555数据手册为准）。
- 直接用比较器输出的“电平”去触发，会在夜间长期保持触发条件，导致：
  - 可能重复触发；
  - 或在某些电路下进入不确定状态。
- 因此必须把“天黑事件”变成一次性短脉冲 TRIG_555，并用二极管钳位防止反向/过冲。

### 3.8 ICM7555触发输入的"标准脉冲整形"电路（比较器输出 → TRIG_555）

在采用"双比较器+RS锁存"后，DARK 的上升沿会非常干净。接下来只需把"上升沿事件"变成ICM7555需要的触发脉冲。

**为什么这样做：**
- ICM7555单稳态需要"短触发"，而不是长时间保持触发条件；
- RC微分把“状态变化（上升沿）”变成“短脉冲”；
- 钳位防止比较器/门电路输出的尖峰把ICM7555输入打到非法电压。

**脉冲宽度怎么选：**
- 脉宽量级由 $\tau = R_{TRIG}C_{TRIG}$ 决定。
- 设计原则：
  1) 脉宽要大于所选ICM7555的最小触发宽度（看数据手册）；
  2) 脉宽要远小于系统2秒响应要求与小时级定时尺度（一般毫秒~百毫秒足够）。

**接口注意：**
- 若比较器为开集电极输出（如LM393类），一定记得加上拉电阻，否则无法形成明确的高电平。
- TRIG_555 节点尽量远离LED功率回路，必要时在TRIG脚加小电容/串阻做EMI抑制。

## 4. 模块6：逻辑控制（一次触发 + 到时锁死 + 天亮复位）

### 4.1 为什么需要 LOCK（解决“夜里到时后又反复亮”的问题）
- 定时结束后，环境仍可能是“天黑”，光控输出 DARK 仍为1。
- 若没有锁死，任何噪声/微小波动都可能再次产生触发，导致反复点亮。
- 因此引入 LOCK：
  - **到时**：置 LOCK=1（夜间锁死）。
  - **天亮**：清 LOCK=0（复位，准备下一晚）。

### 4.2 状态机

> **注**：以下为设计初期的展开式描述。论文最终采用三状态模型（待机 S0 → 工作 S1 → 锁定 S2），日出事件直接从 S1/S2 复位到 S0，无需独立的"复位状态"。

- **S0 白天待机**：DAY=1，LOCK=0，T_ON=0，LED_EN=0。
- **S1 天黑触发/计时照明**：DARK 0→1 产生 TRIG_555，进入计时，T_ON=1 → LED_EN=1。
- **S2 到时关断并锁死**：T_ON 1→0 时置 LOCK=1，LED_EN=0。
- **S3 天亮复位**：DARK 1→0（DAY=1）清 LOCK=0，系统回到 S0。

> 上述 S1/S2/S3 的展开描述等价于论文的三状态模型：S1 对应"工作状态"，S2 对应"锁定状态"，S3 的复位动作直接回到 S0（待机状态）。

### 4.3 逻辑关系

#### 4.3.1 天黑触发脉冲（只在 DARK 上升沿产生）
- 目标：得到一个窄脉冲 TRIG_555（满足555触发电平要求）。
- 建议实现：RC微分/耦合电容 + 电阻泄放 + 二极管钳位整形。
- 触发门控（防止锁死后再触发）：

$$TRIG\_555 = pulse(DARK\uparrow) \land (\lnot LOCK)$$

**为什么一定要“上升沿脉冲”而不是直接用DARK电平：**
- DARK电平在整个夜间都为1，若直接接555 TRIG，理论上会破坏单稳态触发条件（TRIG应为短暂触发）。
- 采用RC微分得到“瞬时脉冲”，能保证“天黑只触发一次”，这与任务书的“误触发问题”直接对应。


#### 4.3.2 到时置锁（只在夜间计时结束瞬间触发）
- 目标：当 T_ON 从1掉到0时，把 LOCK 置1。
- 建议实现：对 T_ON 下降沿做“边沿检测”得到 T_END_PULSE，再去置位锁存。

$$LOCK\_SET = pulse(T\_ON\downarrow) \land DARK$$

**为什么用“T_ON下降沿”置锁：**
- 下降沿代表“定时完成事件”，是最可靠、最直接的“到时信号”。
- 只要捕捉这一次事件，把 LOCK 置1，就能保证接下来整晚都不再响应天黑触发。


#### 4.3.3 天亮复位

$$LOCK\_RST = DAY = \lnot DARK$$

#### 4.3.4 LED 使能（满足“天亮关断”与“定时关闭”）
- 让 LED 只在“夜间且定时有效”亮：

$$LED\_EN = T\_ON \land DARK$$

这么定义的原因：
- **天亮关断**：DARK变0时，不需要等待定时结束，LED_EN立即为0。
- **定时关闭**：T_ON到时变0时，即便仍是夜间，LED_EN也会变0。
- 这条逻辑本身就完成了任务书工作模式中的两条硬约束，把复杂性留给“只触发一次与锁死/复位”。

- 若还要叠加欠压保护/总使能，可在此再与 V_BAT_OK 相与（不属于4/5/6，但接口兼容）。

### 4.4 LOCK 的电路实现选型
- **CMOS 锁存/触发器（带独立置位/复位）**
  - 置位：LOCK_SET（到时脉冲）
  - 复位：LOCK_RST（DAY）
- 选型原则：静态电流低、输入容限宽、Mutisim库易仿真。
