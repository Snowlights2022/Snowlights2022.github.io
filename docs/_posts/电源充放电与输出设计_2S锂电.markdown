# 电源充放电与输出设计（12V太阳能板 + 2S/3S锂电串联）

> 前提：12V“标称”太阳能板通常 Vmp≈17–18V、Voc≈21–22V；2S锂电满充 8.4V，3S锂电满充 12.6V。
> 因此**不能**用“太阳能板直连电池”的方式充电，必须采用降压型充电/限压限流电路。

## 0. 任务书指标与本方案的对应关系
- 任务书给出的“3.7V锂电或6V铅酸”是常见低压方案；你当前改为 2S/3S 锂电后：
  - 光控/时控/低功耗逻辑不变；
  - 充放电与保护需要升级（多串锂电必须有均衡与过充/过放保护）。

## 1. 系统电源模块划分
1) 太阳能输入保护与防反灌（PV_IN）
2) 太阳能升降压充电（BuckBoost）
3) 电池组（2S）+ BMS保护（过充/过放）
4) 负载电源路径与开关（低边NMOS或理想二极管路径）
5) 控制电路稳压
6) LED驱动输出（高效率开关恒流）

## 2. PV输入与防反灌（必须）
**目的**：夜间防止电池通过太阳能板反向放电；抑制浪涌；提升可靠性。
- PV+ 串联肖特基二极管（简单但有压降与损耗）。

## 3. 太阳能对多串锂电的充电（核心：Buck + CC/CV）

### 3.1 充电目标电压
- 2S：$V_{CHG}=8.4V$（每节4.2V）

### 3.2 为什么需要BuckBoost
- PV面板电压可能高于或者低于电池。
- 锂电充电必须遵循：先恒流（CC）→ 到达目标电压后恒压（CV）→ 电流衰减到截止。
- 仅靠电阻会效率低、发热大、充电不可控且不安全。

### 3.3 工程实现路线

**BuckBoost + 电流采样限流**
- 优点：Multisim里更容易用等效模型搭出来。
- 风险：细节不当会导致充电过压/不稳定；实物要谨慎验证。

### 3.4 充电电流如何定
- 先由电池容量 C_Ah 确定充电倍率：常用 0.2C–0.5C（更安全可取0.2C–0.3C）。
- 例：2S1P 2200mAh，取0.3C→约0.66A。
- PV功率要能覆盖：$P\approx V_{CHG}\cdot I_{CHG}$（再考虑Buck效率）。

## 4. 2S电池组保护
多串锂电必须具备：
- 过充保护（单节>4.2V附近切断）
- 过放保护（单节<3.0V附近切断，任务书强制项）
- 过流/短路保护
- **均衡**（防止串联电芯长期不一致导致某一节先过充或先过放）

工程上最简单可靠的做法：
- 选用现成2S BMS作为电池包保护板。

- BMS保证单节电压受控，提高寿命与安全性；
- 均衡是“串联锂电”区别于单节锂电的关键。

## 5. 负载供电路径与低功耗门控
- 电池组输出经BMS后得到 PACK+ / PACK-

## 6. 输出：LED驱动（高效率≥85%）

### 6.1 为什么不建议电阻限流
- 2S电池电压变化范围大，电阻限流会导致亮度随电压大幅变化且效率低。

### 6.2 推荐：Buck恒流LED驱动
- 以PACK电压为输入，Buck恒流输出到LED（1–3W）。
- 优点：效率高、发热小、亮度稳定，容易满足≥85%。
- LED使能信号由控制逻辑提供（天黑且定时有效）。

## 7. 仿真与实物落地建议
- Mulitisim里先用“等效电源”替代复杂充电芯片，验证控制逻辑与功耗门控；
- 实物阶段再落地具体充电/驱动器件。
